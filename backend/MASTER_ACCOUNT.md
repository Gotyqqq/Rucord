# Мастер-аккаунт

Мастер-аккаунт — это пользователь с полными правами на **всех** серверах: может заходить на любой сервер, редактировать и удалять любые сообщения, управлять ролями, банить и т.д.

## Как назначить мастер-аккаунт

Только владелец сервера (тот, у кого есть доступ к файлам бэкенда) может выдать права мастера. Через интерфейс приложения это сделать **нельзя** — так никто не сможет «украсть» права.

1. Откройте файл `backend/.env`.
2. Добавьте строку (или отредактируйте существующую):
   ```env
   MASTER_USER_IDS=1
   ```
   Здесь `1` — это **ID вашего пользователя** в базе (у первого зарегистрированного пользователя обычно id = 1).
3. Чтобы назначить несколько мастеров, перечислите ID через запятую:
   ```env
   MASTER_USER_IDS=1,5,12
   ```
4. Перезапустите бэкенд.

## Как узнать свой user id

- Посмотрите в ответе API `GET /api/auth/me` поле `user.id`.
- Или в базе `backend/rucord.db` в таблице `users`.

## Поведение мастера

- В списке серверов отображаются **все** серверы, не только те, куда вы вступили.
- При первом заходе на сервер мастер автоматически добавляется в участники с ролью «Мастер» (полные права).
- Мастер может редактировать и удалять любые сообщения, управлять каналами, ролями, банить и т.д. на любом сервере.
