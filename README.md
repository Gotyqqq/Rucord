# Rucord

Упрощённый веб-аналог Discord: серверы, текстовые и голосовые каналы, личные сообщения, роли и права, реакция и вложения.

Документация по архитектуре и внутреннему устройству — в **[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)**. Её стоит обновлять при изменении структуры проекта, API или сокетов.

---

## Требования

- **Node.js** (LTS), npm

Рекомендуется установить Node.js с https://nodejs.org/ (LTS). Проверка:

```bash
node --version
npm --version
```

---

## Запуск (локально)

### 1. Установка зависимостей

```bash
cd backend
npm install
```

```bash
cd frontend
npm install
```

При проблемах с нативными модулями (если появятся) на Windows может понадобиться:  
`npm install -g windows-build-tools` (PowerShell от администратора).

### 2. Запуск бэкенда

В папке `backend`:

```bash
npm start
```

Сервер: **http://localhost:3001**, API: **http://localhost:3001/api**.

### 3. Запуск фронтенда

В папке `frontend` (второй терминал):

```bash
npm run dev
```

Интерфейс: **http://localhost:5173**.

### 4. Открыть приложение

В браузере: **http://localhost:5173**.

## Как пользоваться

- **Регистрация / вход** — на лендинге.
- **Сервер:** кнопка **+** в левой панели → создать сервер. Появляются канал «общий» и роли.
- **Приглашение:** кнопка «Пригласить» → скопировать инвайт-код.
- **Присоединиться:** кнопка со стрелкой (↗) → ввести инвайт.
- **Чат:** выбрать сервер → канал → ввод сообщения внизу (Enter или ➤). Поддерживаются вложения, реакции, упоминания, slowmode.
- **Голос:** войти в голосовой канал из списка каналов; микрофон/звук — кнопки в панели голоса. Настройки голоса и проверка микрофона — в настройках пользователя (иконка шестерёнки).
- **Управление сервером (владелец):** ⚙ у названия сервера → Общие / Роли / Участники / Каналы и т.д.

## Технологии

| Часть        | Стек              | Назначение                    |
|-------------|-------------------|-------------------------------|
| Сервер      | Node.js, Express  | HTTP API, раздача статики     |
| Реалтайм    | Socket.IO         | Сообщения, типинг, голос (сигналы) |
| БД          | SQLite (sql.js)   | Пользователи, серверы, каналы, сообщения, роли, муты, баны |
| Фронтенд    | React             | SPA                           |
| Сборка      | Vite              | Dev-сервер, прод-сборка, worklet |
| Голос       | WebRTC (P2P)      | Аудио между участниками       |
| Авторизация | JWT, bcrypt       | Вход, защита маршрутов        |

## Частые проблемы

- **«node/npm не найден»** — установить Node.js LTS и перезапустить терминал.
- **Порт 3001 или 5173 занят** — закрыть другое приложение на этом порту или поменять порт в `backend/server.js` / `frontend/vite.config.js`.
- **CORS** — убедиться, что бэкенд запущен на 3001, фронт на 5173.
- **Данные** хранятся в `backend/rucord.db`; папку `backend/uploads` не удалять.
